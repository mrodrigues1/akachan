# Internationalization (i18n) Implementation Plan — Overview

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement each task plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extract every user-facing string in Akachan into Android string resources, ship English + Brazilian Portuguese, add an in-app language picker, and lint-guard against hardcoded-string regressions.

**Architecture:** Pure resource + UI work — no domain/module change. Base copy lives in `values/strings.xml` (English); `values-pt-rBR/strings.xml` holds the translation. Per-app locale uses the AppCompat backport so it works on minSdk 26. Lint enforces no hardcoded text once both locales are complete.

**Tech Stack:** Android string resources (`<string>`, `<plurals>`), Jetpack Compose `stringResource` / `pluralStringResource`, AGP 9.1.0 `androidResources.localeFilters`, `androidx.appcompat` `AppCompatDelegate.setApplicationLocales`, Android Lint.

**Spec:** [`docs/superpowers/specs/2026-06-17-i18n-localization-design.md`](../specs/2026-06-17-i18n-localization-design.md)

**Linear project:** [Internationalization (i18n)](https://linear.app/akachan/project/internationalization-i18n-bd75c873c822)

## Global Constraints

- **Languages:** English (base, `values/`) + Brazilian Portuguese (`values-pt-rBR/`). No other locales.
- **Key naming:** `<feature>_<element>` snake_case, grouped under an XML comment header per feature. Reuse existing generic keys (`cancel`, `delete`, `edit`, `back`, `try_again`, `loading`, `more_options`) — never duplicate common verbs.
- **No concatenation:** counts → `<plurals>` (`quantity="one"`/`"other"`); durations/dates/volumes/names/counts → positional format args (`%1$s`, `%2$d`). Never build user-facing text with Kotlin `+`.
- **`contentDescription`:** extract all meaningful a11y strings; leave decorative images `null`.
- **ViewModels stay locale-agnostic** where practical: resolve copy in the composable via `stringResource`, or expose string resource IDs in UI state. Use an injected `@ApplicationContext Context` only when a non-Compose surface (receiver, notification manager) needs the string.
- **Min SDK 26, Compile SDK 36, AGP 9.1.0, Kotlin 2.3.20.** JVM 17.
- **No new remote integrations.** `androidx.appcompat` (a UI library) is the only new dependency, added in the final issue.
- **Validation per task:** `./gradlew ktlintFormat` (auto via pre-commit), `./gradlew lintDebug`, `./gradlew assembleDebug`, and the relevant `./gradlew test --tests "..."`. Full `./gradlew build` before the PR.

## The extraction recipe (issues 2–7)

Every extraction issue follows the same loop per file. It is mechanical and
behaviour-preserving:

1. **Find literals.** In the target file, locate user-facing string literals:
   `Text("...")`, `contentDescription = "..."`, `label = { Text("...") }`,
   `placeholder`, `title`, dialog/snackbar text, and any
   `"..." + value` / `"$value ..."` concatenations.
2. **Choose a key.** `<feature>_<element>`. Check `strings.xml` first — reuse a
   generic key if one fits.
3. **Add to base `strings.xml`** under the feature's comment block. For counts use
   `<plurals>`; for interpolated values use positional args.
4. **Replace the literal** with `stringResource(R.string.key)` /
   `stringResource(R.string.key, arg)` / `pluralStringResource(R.plurals.key, n, n)`.
   Add `import androidx.compose.ui.res.stringResource` (and
   `pluralStringResource`) as needed.
5. **For ViewModel-embedded copy:** move the literal into `strings.xml` and either
   resolve it in the composable, or expose a `@StringRes Int` in UI state. Only
   inject `Context` when no composable is in the resolve path.
6. **Verify** (commands in each task). Update the lint baseline so it shrinks.
7. **Commit** with `feat(i18n): extract <area> strings to resources`.

## Definition of done (whole project)

- `./gradlew lintDebug` reports **zero** `HardcodedText` and zero
  `MissingTranslation` for `en` + `pt-rBR`; lint runs `fatal` (issue 9).
- App runs end-to-end in both languages; language switches live and survives
  process restart.
- Lint baseline file deleted (count reached zero).
- `./gradlew build` green.
